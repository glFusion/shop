glFusion Shop Plugin Configuration

Main Shop Options

Administrator's Email Address
Enter the email addresses to which confirmation email messages should be sent. If this is empty, the site email address will be used.
Currency
This is the currency in which you want all transactions to occur. Your Payment Provider account must have the choosen currency enabled.

This plugin currently requires you to have only one currency enabled. All the currencies listed in ISO 4217 are available for selection, If the selected currency is not supported by a payment gateway such as PayPal the gateway will be disabled.

Anonymous users can buy?
Select "Yes" if you allow anonymous users to buy items from your site. Select "No" to allow only registered users to purchase. Access to specific groups can also be granted by category.
Add to main menu?
Select "Yes" to have the plugin added to the "Extras" menu. Select "No" if you wish to manually add this plugin to another menu.
Default sort order for product display
Select the default sort order for products shown on the main product list.
Max products displayed per page
Set the maximum number of products to display on a single page.
Catalog Columns
Set the number of columns for the catalog list, for block-style templates.
Include plugins in catalog?
Select "Yes" to include products provided by other plugins in the catalog. Plugin products will appear at the end of the first page, and are not included in search results.
Enable Comments?
Select whether you'd like to enable user comments for products.
Enable Product Ratings?
Select whether you'd like to enable user ratings for products.
Anonymous can rate products?
Select whether you'd like to allow anonymous users to rate products. If this is "No" then only logged-in users can give ratings.
Display glFusion Blocks
Select whether to display left blocks, right blocks, both, or neither when the product catalog is displayed.
Centerblock?
Select whether the Shop plugin should take over the home page as a centerblock.
Unit of Weight Measurement
Select your preferred unit of measure for product weight: Kilograms or Pounds.
Unit of Size Measurement
Select your preferred unit of measure for sizes: Inches or Centimetres.
Link to Terms and Conditions
Enter an optional URL to your TOC page for purchases. This will appear on the cart form prior to checkout.
Days to Purge Carts
Enter a number of days after which abandoned shopping carts will be deleted. Zero disables purging of carts.
Days to Purge Pending Orders
Enter a number of days after which abandoned orders, those that have not been submitted for payment, will be deleted. Zero disables purging of old orders.
Years before redacting data
A number of years before personal information like addresses will be redacted for deleted users. Default is "7", your regional laws may require a different value.
Enable shopping cart?
Select whether the internal shopping cart should be enabled. If this is not enabled, there will be no way for a buyer to purchase items with various attributes, as those items must go through the shopping cart and checkout process.

Image and Path Settings

Max. Thumbnail Dimension
Enter the maximum size, in pixels, for thumbnail images. This is for efficiency only and should be set to a reasonable value, but large enough for any thumbnails to be properly rendered. The thumbnail will be sized by the template so that its longest side does not exceed this value while preserving the aspect ratio.
Full Path to Downloadable Files
This is the full path, including trailing slash, where files will be stored for later download by the buyer.

The default path is created under the private/data/shop directory during plugin installation. This path may be changed, but must be readable by your web server. If you wish to be able to upload product files via the product form, then it must also be writable.

Max size for downloadable files
This is the maximum size, in MB, of files that will be downloadable by buyers. This number actually limits the size of file uploads, not downloads.
Temporary Working Directory
Enter the full path to a working directory to be used during the creation of encrypted buttons. The web server must be able to create and delete files in this directory.

When buttons are encrypted, several temporary files are created and then deleted. These do not contain sensitive information, but this directory should still be fairly private.

New Product Defaults

These options set the default values for new products and can be overridden for each product.

Default Product Type
Select the type of product sold most often. This will be set by default when new products are created.
Product Enabled?
If "Yes", then the "Enabled" checkbox will be checked automatically for new products.
Product is Taxable?
If "Yes", then the "Taxable" checkbox will be checked automatically for new products.
Product Featured?
If "Yes", then the "Featured" checkbox will be checked automatically for new products.
Default Expiration (days)?
Enter the default number of days after a purchase for a downloadable product to be available.
Track Qty On Hand?
Select "Yes" to have the plugin track the quantity on hand of an item, by default. The administrator can update the quantity by editing the product, and the quantity will be reduced as purchases are made (to a minimum of zero).
Action when Qty On Hand = 0
Select the action to be taken when the quantity on hand reaches zero. Options are:
  • Allow Backordering: Continue to allow sales of the product.
  • Hide from Catalog: Hide the product from the catalog so it cannot be seen or purchased.
  • Keep in Catalog, Prevent Sales: Continue to display the product in the catalog but remove the purchase links.
Product Detail Template Version

Select the default template version to use for the Product Detail views. Two template versions are currently included: the previous template and a new version that emphasizes product images. This is currently a global setting and affects all products but a future version may include per-product selections.

You may add your own templates by creating a directory under private/shop/templates/detail and creating product_detail.thtml and product_detail_attrib.thtml. Both files are required (this may change later).

Homepage Layout
Select whether the Shop homepage will list all products, or categories. NOTE: If "Category List" is selected, the Home category is not shown.
Enable Service Functions?
Select "Yes" to masquerade as the Paypal plugin by enabling the service functions other plugins to interact with Shop. This is a temporary option for compatibility when updateing from the PayPal plugin to Shop and will be removed later. IMPORTANT: Be sure that the Paypal plugin is removed or disabled before enabling this option.
Enable Public Access?
Select "Yes" to make the shop publically accessible. Selecting "No" prevents anyone except site administrators from accessing the plugin. Use during testing. You must clear the cache after changing this setting.
Enable Catalog?
Select whether the catalog is enabled. This is normally "Yes", but if you're only using the Shop to support other plugins you should probably set this to "No".
IPN URL Override?
If your site does not support SSL you may have to serve the IPN handlers from another URL, such as "https://yourhost.com/~yourname/ipn". If so, enter that URL here, including the path to where you will place the IPN handlers. You may need to modify the handlers themselves to include lib-common.php from the correct path as well.
SKU as Product ID
Select this option to use the product SKU (Name) values as the ID in URLs. This may improve SEO and will make the URLs easier to understand. This option requres that each product has a unique SKU and will be reset to "No" if duplicate SKUs are found.
Default Admin View
Select the list shown by default when accessing the Administation area.
Enable Fast-Checkout
Select "Yes" to enable skipping directly to final checkout if the order meets certain requirements.
Starting Invoice Number
Invoice numbers are created sequentially, beginning with "1", as orders are finalized. If numeric, this number will be added to the sequence number when orders are displayed. If not, it will be used as a formatting string, i.e. you could enter "INV-%d".

Block Settings

Number of products in Random block
This is the number of products that will be shown in the "Random Product" block. If this number is less than one, then one product will be shown.
Number of products in Featured block
This is the number of products that will be shown in the "Featured Product" block. If this number is less than one, then one product will be shown.
Number of products in Popular block
This is the number of products that will be shown in the "Popular Products" block. If this number is less than one, then one product will be shown.

Debugging

Log Level?
Select the minimum level of messages to be logged to the system log. "Debug" includes debugging IPN data which may contain PII.

Checkout Address Collection

Street Address
At checkout, the Street Address field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
City
At checkout, the City field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
State
At checkout, the State field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Country
At checkout, the Country field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Remit To
(optional) Enter a string to appear on remittance tickets and invoices, e.g. "Attn: Accounts Receivable"
Postal Code
At checkout, the Postal Code field can be Required, Optional, or Not Available for the Billing and Shipping addresses (whichever are used).
Phone Number
Indicate if the buyer must enter a telephone number at checkout. Entry may be required, optional, or not available.
Address Validator
Select an available API to validate user-entered addresses. You will need to set up an acount with the provider and enter credentials. The USPS provider credentials are configured through the Shipping module. TaxCloud is configured with the Tax Provider information.
SmartyStreets ID
If you elect to use smartystreets.com as your address validator, enter your user ID here.
SmartyStreets ID
If you elect to use smartystreets.com as your address validator, enter your token here.
SmartyStreets License
Enter the license or subscription type for SmartyStreets. The default is for the free subscription.

Shop Information

Remittance/Support Name
Enter the name of a person or department to receive remittances or to contact for support.
Company Name
Enter the name of your store. During installation this was set to the site name but can be changed here. This will be displayed on remittance coupons and receipts.
Shop Address Line 1
Enter the street address for your shop. This value is optional, but should be provided to appear on orders and especially if you accept payments by check.
Shop Address Line 2
Enter the second line for your address, if any.
Shop City
Enter the city where your shop is located.
Shop State
Enter the ISO code (2-letter abbr.) for the state or province where your shop is located.
Shop Zip
Enter the postal code where your shop is located. This is used to determine the shipping origin.
Shop Phone
(Optional) Enter the phone number for your shop to appear on invoices.
Shop E-Mail
(Optional) Enter the email address for your shop to appear on invoices.
Shop Country
Select the primary country code for the shop's location. This is used to format international addresses.
Purge Sale Prices?
Enter a number of days to keep sale prices in the database, or "-1" to disable purging. This is useful if you wish to re-activate periodic sales by changing the dates.
Enable Gift Cards?
Select "Yes" to enable gift cards. Gift cards are created as normal products.
Default Gift Card Expiration
Enter the default number of days after which a promotional gift card will expire, or 0 fo no expiration. This is only for gift cards sent as a thank-you or loyalty award. Purchased gift cards never expire.
Letters to use in Gift Card Codes
Select the type of letters to be used when creating gift card codes- uppercase, lowercase, both, or neither.
Use Numbers in GC Codes?
Select whether to include numbers in gift card codes.
Use Symbols in GC Codes?
Select whether to include symbols and punctuation in gift card codes.
Prefix for Coupon Codes
Enter any short prefix that should be prepended to the front of all coupon codes.
Suffix for Coupon Codes
Enter any short suffix that should be appended to the end of all coupon codes.
Length for Coupon Codes
Enter the length for the coupon codes. This is ignored if a Mask string is configured below.
Mask for Coupon Codes
Enter an optional mask for the coupon code format. The letter "X" will be replaced with a generated character while any other character will be include as-is. Example: XXX-XXX-XXX.
Sales Tax Nexuses
Create elements for each location where you have a nexus for sales tax purposes. Enter values in ISO-3166-2 format (CC-SS where "CC" is the country code and "SS" is the state or region). For example, use "US-CA" for California, USA. If the state/region is omitted then the nexus will apply to the entire country. If empty, sales tax will always be calculated.
Sales Tax Provider
Select the provider for sales tax calculations. "Internal" uses the internal tax tables, "Configured Rate" applies the rate set below to all taxable items.
Tax Rate
If you select the "Configured Rate" option, then enter the percentage charged by your shop as sales tax or zero if you do not charge tax. This will be calculated and added to orders for taxable items. Enter the rate as a decimal, e.g. ".085" for 8.5%.
Nexus for virtual products
Select how the tax rate for virtual products is calculates. Options are to use the origin (Shop) location, use the customer-entered shipping address, or use the automatically-determined location via IP geolocation (if enabled).
Affiliate URL Key
The key name used in the referral link. Default is "shop_ref", so the link would appear as "https://sitename/?shop_ref=xxxxx". If you change this be sure it is a unique value not used elswhere in glFusion.
Affiliate Cookie Expiration
Enter the number of days for which a referral is valid if no items are added to the buyer's cart.
Affiliate Cart Expiration
Enter the number of days for which the referral is valid once items have been added to the buyer's cart.

Affiliate Program Settings

Affiliate Program Enabled
Select whether to enable the affiliate referral program globally.
Allow affiliate code entry during checkout
Select whether an affiliate code can be entered during checkout to override a referral created from an affiliate link
Expiration in days for referrals
Enter the number of days after a referral is made when it will no longer be used for an order. Referrals which have resulted in products being added to a cart will expire when the cart expires.
Default Affiliate Bonus Percentage
Enter the default bonus percentage of net sales to be paid to affiliates. Use a whole number, e.g. "4.5" for 4.5% (.045).
Minimum order status
Select the minimum order status to be considered for affiliate payouts. Orders that have not yet reached this status will not have bonuses paid.
Minimum Affiliate Payment
Enter the minimum amount, in the Shop's currency, for affiliate payments. Affiliates whose balance has not reached this level will not receive a payment when payments are processed.
Affiliate URL key
This is the url key used to detect an affiliate ID. You may change this to something else but be sure it cannot be used for any other purpose on your site.

Geocoding Settings

Geolocation Provider
Select a geolocation provider, if any. You will need to sign up with the provider and obtain an API key to enter below.
IPStack API Key
Enter the API key provided by ipstack.com if this provider is used.
IPGeo API Key
Enter the API key provided by ipgeolocation.io if this provider is used.
Default Google Product Category
Enter the code for the default Google product category to be used in feeds, where a category does not have a code specified.
Link anonymous orders to new user account
Select whether anonymous orders with a matching e-mail address will be automatically linked to a new user's account at registration.
